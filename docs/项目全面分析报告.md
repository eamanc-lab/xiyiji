**三种处理模式:**

| 模式 | 条件 | 流程 | 置信度 |
|------|------|------|--------|
| Streaming | 后端支持 processAudioStream | 边推理边播放, 低首帧延迟 | 100% |

## 十一、核心数据流

### 自动直播闭环

```
弹幕事件(6平台) → EventBatcher(4s窗口) → AiLoopService
    → QwenService(Prompt构建 + 批量生成 10-15条)
    → QueueManager(pending → ready → playing → buffered → done)
    → TtsService(UCloud 异步合成, >600字分段)
    → LivePipelineService(单任务排他提交)
    → PipelineService(Streaming / Parallel / Single-shot)
    → yundingyunbo(GPU推理, NDJSON协议)
    → Player(chunk 流式播放)
    → IPC chunk-played 回调 → 标记完成 → 下一条
```

### 背压控制

| 层级 | 机制 | 置信度 |
|------|------|--------|
| AI 生成 | ready < 3 才触发补充 | 98% |
| TTS | 并发限制 | 95% |
| 管线 | 同一时间仅 1 个任务 | 100% |
| 播放器 | LIVE_MAX_BUFFERED_CHUNKS = 4 | 100% |
| 队列 | 120s 过期清理 | 100% |

### 直播间启动流程

```
UI "开始直播" → live:room-start IPC
  → 验证 License
  → 加载方案（形象视频 + TTS 语音 + VAD 参数 + 绿幕）
  → 激活后端（yundingyunbo / electron player）
  → 打开播放器窗口
  → initAvatar()
  → 启动: eventBatcher, aiLoopService, livePipelineService, roomSession
  → 设置禁词/黑名单/弹幕过滤
  → 启动 License 心跳
  → 返回 { ok, avatarVideoPath, mediaType, rendererMode }
```

### 弹幕打断流程

```
弹幕事件 → EventBatcher → AiLoopService.receiveBatch()
  → 检测关键词 → 切换商品链接
  → 带弹幕上下文生成新话术（focused prompt, 2条）
  → insertAfterCurrent() 插入队列头部
  → 优先播放
```
